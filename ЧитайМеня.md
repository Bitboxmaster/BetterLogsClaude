# BetterLogsClaude

Автоматическое сохранение сессий Claude Code в читаемые Markdown-файлы с синхронизацией в Obsidian через iCloud.

## Что делает

После каждого ответа Claude Code хук автоматически конвертирует текущую сессию из JSONL в Markdown и сохраняет в папку Obsidian. Файлы именуются по дате и первому вопросу: `2026-03-22 Оптимистичная блокировка.md`.

## Файлы

| Файл | Назначение |
|------|------------|
| `settings.json` | Конфиг Claude Code с хуком на событие `Stop` |
| `convert.py` | Скрипт конвертации JSONL → Markdown |
| `logs.conf` | Все пути в одном месте — менять только здесь |

## Установка

### 1. Скопировать файлы в `~/.claude/`

```bash
cp settings.json ~/.claude/
cp convert.py ~/.claude/
cp logs.conf ~/.claude/
```

### 2. Настроить пути

Открой `~/.claude/logs.conf` и измени пути под себя:

```bash
# Папка с JSONL сессиями (обычно менять не нужно)
JSONL_DIR="$HOME/.claude/projects"

# Папка для Markdown логов — ИЗМЕНИ НА СВОЮ
LOGS_DIR=$HOME'/Library/Mobile Documents/iCloud~md~obsidian/Documents/Logs/claude-logs'

# Путь к скрипту (обычно менять не нужно)
CONVERT_SCRIPT="$HOME/.claude/convert.py"

# Задержка перед конвертацией
SLEEP_SEC=3
```

> **Важно:** если путь содержит пробелы, используй формат `$HOME'/path with spaces/folder'`

### 3. Создать папку для логов

```bash
mkdir -p ~/Library/Mobile\ Documents/iCloud~md~obsidian/Documents/Logs/claude-logs
```

### 4. Проверить

Запусти вручную:

```bash
. ~/.claude/logs.conf && LATEST=$(find "$JSONL_DIR" -name '*.jsonl' -type f -exec stat -f '%m %N' {} + 2>/dev/null | sort -rn | head -1 | cut -d' ' -f2-) && [ -f "$LATEST" ] && python3 "$CONVERT_SCRIPT" "$LATEST" "$LOGS_DIR"
```

Должен появиться `✅ /путь/к/файлу.md — N сообщений`.

## Как работает

1. Claude Code завершает ответ → срабатывает хук `Stop`
2. `sleep 3` — ожидание записи JSONL на диск
3. `find` находит самый свежий JSONL-файл сессии
4. `convert.py` парсит JSONL, извлекает сообщения, фильтрует служебные команды (`/model`, `/color` и т.д.)
5. Имя файла генерируется из даты + первого вопроса пользователя
6. Если файл с таким именем уже есть — перезаписывается (обновление текущей сессии)
7. Markdown появляется в Obsidian через iCloud

## Конвертация старых сессий

Для массовой конвертации всех существующих JSONL:

```bash
shopt -s globstar
. ~/.claude/logs.conf
for f in "$JSONL_DIR"/**/*.jsonl; do
  python3 "$CONVERT_SCRIPT" "$f" "$LOGS_DIR"
done
```

## Формат выходного файла

```markdown
# Оптимистичная блокировка
**Дата:** 2026-03-22

---

## 👤 Ты (10:28:04)

Расскажи про оптимистичную блокировку

---

## 🤖 Claude (10:28:12)

Оптимистичная блокировка — это...

---
```

## Возможные проблемы

| Проблема | Решение |
|----------|---------|
| Пустые файлы (нет ответа Claude) | Увеличь `SLEEP_SEC` в `logs.conf` |
| Хук не срабатывает | Проверь `cat ~/.claude/settings.json` — хук должен быть в `Stop` |
| Ошибка с путями | Проверь `logs.conf` — пробелы в путях требуют `$HOME'/path'` |
| Служебные команды в логах | Обновить `convert.py` — фильтр `<local-command` |
| Дубликаты файлов | Скрипт перезаписывает файл с тем же именем |

## Требования

- macOS (используется `stat -f` для поиска свежих файлов)
- Python 3
- Claude Code CLI
- Obsidian + iCloud (опционально — можно писать в любую папку)
